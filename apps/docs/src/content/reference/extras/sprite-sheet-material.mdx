A component for static or animatable spritesheets.

<Example path="extras/sprite-sheet-material" />

## Basics

Threejs's `Sprite` is a plane that always face the camera and cannot cast
shadows. The `SpriteMaterial` is transparent by default, isn't affected by
lighting and generally has a single static texture. All of that can be a
drawback so using a normal `Mesh`, `PlaneGeometry` and `MeshStandardMaterial`
is a valid option.

## SpriteSheets

A common pattern in 2D graphics is to pack sprites into one texture called a
spritesheet.

### Multiple static Sprites

The sheet data for the sprite frames within the texture can be given using the `data` prop

The fruit example:

```svelte
<script lang="ts">
  import { T } from '@threlte/core'
  import { SpriteSheetMaterial, type SpriteSheetData } from '@threlte/extras'

  const data = {
    frames: [
      {
        x: 0,
        y: 48,
        width: 16,
        height: 16,
        name: 'apple_red'
      },
      {
        x: 16,
        y: 16,
        width: 16,
        height: 16,
        name: 'pear_orange'
      },
      {
        x: 32,
        y: 0,
        width: 16,
        height: 16,
        name: 'grapes_green'
      }
    ]
  } satisfies SpriteSheetData

  const selectedSprite = 'apple_red'
</script>

<T.Mesh>
  <T.PlaneGeometry />
  <SpriteSheetMaterial
    textureUrl="/textures/sprites/fruit.png"
    {data}
    {selectedSprite}
  />
</T.Mesh>
```

### Animated Sprites

The coin

...and the knight

This material is most easily used by passing it a [spritesheet](https://www.aseprite.org/docs/sprite-sheet/) URL and a [JSON metadata file](https://gist.github.com/dacap/db18e5747a4b6e208d3c) URL.

Currently, JSON metadata using [Aseprite's](https://www.aseprite.org/) [hash export format](https://www.aseprite.org/docs/cli/#data) is supported.

Animation names [from tags](https://www.aseprite.org/docs/tags) can be used to transition to specific animations in the spritesheet.

```svelte {4,5}+
<T.Sprite>
  <SpriteSheetMaterial
    animation="Idle"
    textureUrl="./player.png"
    dataUrl="./player.json"
  />
</T.Sprite>
```

If no metadata file is provided, additional props must be passed to run an animation:

- `totalFrames`, if the spritesheet is only a single row.
- `totalFrames`, `rows`, and `columns`, otherwise.

```svelte {4-6}+
<T.Sprite>
  <SpriteSheetMaterial
    textureUrl="./fire.png"
    totalFrames={14}
    rows={4}
    columns={4}
  />
</T.Sprite>
```

Additionally, if a sheet with no JSON supplied has multiple animations, start and end frames must be passed to run an animation within the sheet.

```svelte {7-8}+
<T.Sprite>
  <SpriteSheetMaterial
    textureUrl="./fire.png"
    totalFrames={14}
    rows={4}
    columns={4}
    startFrame={4}
    endFrame={8}
  />
</T.Sprite>
```

`<SpriteSheetMaterial>` can be attached to a `<T.Sprite>` as well as a `<T.Mesh>`.

<Example path="extras/animated-sprite-mesh" />

In the case of a `Mesh` parent a `MeshBasicMaterial` will be used by default, instead of a `SpriteMaterial` when attached to a `Sprite`. A custom depth material will be attached when parented to a mesh to support shadows.

Any other material type can be used as well.

```svelte {3}+
<T.Mesh>
  <SpriteSheetMaterial
    is={THREE.MeshStandardMaterial}
    textureUrl="./fire.png"
    totalFrames={14}
  />
</T.Mesh>
```
